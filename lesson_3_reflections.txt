Quando gostaria de usar um repositório remoto em vez de manter o trabalho local?

  Caso esteja desenvolvendo um projeto em grupo ou para ter acesso em qualquer lugar e em qualquer dispositivo.
  
Descreva as diferenças entre bifurcações, clones e branches. Quando você usaria um e não os outros?

  Clone: Cria uma cópia do repositório no GitHub no computador local. Branches: Cria um novo ramo a partir de um commit, sendo que esse     não interfere no ramo original daquele commit. Bifurcações: Cria uma cópia do repositório no GitHub de algum autor no seu repositório do   GitHub, no qual você pode realizar suas próprias alterações conforme desejar, podendo, inclusive, clonar para o computador local.
  
Qual é o benefício de ter uma cópia do último estado conhecido do remoto armazenado localmente?

  Evita conflito de alterações realizadas no mesmo arquivo em diferentes locais. 
  
Como você colaboraria sem usar Git ou GitHub? O que seria mais fácil e mais difícil?

  Usaria algum método adotado mundialmente pelos melhores programadores. Talvez a falta de algumas funções existentes no Git e GitHub       poderia dificultar o trabalho conjunto de programadores. Seria mais fácil por não exigir um conhecimento abrangente dessas técnicas
  presentes no Git e GitHub.
  
Quando você gostaria de fazer alterações em um branch separado, e não diretamente no master? Quais são os benefícios de cada abordagem?

  Quando estivesse trabalhando em um projeto com uma equipe de programadores ou para testar novas ferramentas. Ao criar um novo branch,     você tem pode fazer alterações nos arquivos sem interferir no ramo original. Caso contrário, se você encontrar algum bug, terá de         aplicá-lo no ramo principal.
